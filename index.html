    Pregunta 1
    ¿Qué datos necesita el sistema para funcionar?
    El sistema maneja datos de tres fuentes principales:
    • Catálogo de lugares: nombre, categoría, coordenadas GPS reales, descripción multiidioma, horarios, rango de precios, calificación y reseñas.
    • Perfil del usuario: tipo de viajero, idioma, presupuesto, historial de lugares visitados y preferencias de categoría.
    • Datos operativos: métricas de uso como lugares más consultados, horarios pico y perfiles predominantes. Invisibles para el usuario pero clave para decisiones de negocio.
    Pregunta 2
    ¿Dónde se podrían almacenar esos datos?
    • Prototipo actual: JSON local en el HTML — sin backend, funciona offline.
    • Base de datos principal: PostgreSQL + PostGIS para consultas geoespaciales reales.
    • Perfiles de usuario: Firebase Auth + Firestore para sesiones persistentes.
    • Caché: Redis para respuestas instantáneas en búsquedas frecuentes.
    • Backups: copias automáticas diarias en almacenamiento frío (AWS S3).
    Pregunta 3
    ¿Cómo ayudaría la IA o el análisis de datos?
    • Recomendaciones automáticas: si muchos turistas visitan el Malecón y luego Rancho y Leña, el modelo aprende ese patrón y lo sugiere al siguiente usuario similar.
    • Itinerarios inteligentes: el usuario ingresa días y presupuesto y la plataforma genera un recorrido optimizado automáticamente.
    • Panel de analítica: dashboard interno con lugares más visitados, perfiles predominantes y picos de demanda por temporada.
    • Alertas de calidad: detección automática de reseñas negativas antes de que se queje un cliente directamente.
    • Búsqueda en lenguaje natural: "quiero almorzar barato con mis hijos cerca del parque" — el sistema lo interpreta solo.
    Pregunta 4
    ¿Qué riesgo de privacidad podría existir?
    • Geolocalización sin consentimiento: acceder al GPS sin opt-in viola la Ley 1581/2012 de Colombia y el GDPR europeo.
    • Perfiles sensibles: combinar edad, perfil y rutas puede revelar condición médica, creencias o situación económica sin que el usuario lo sepa.
    • Terceros: los tiles de OpenStreetMap y las APIs de Google reciben metadatos de navegación del usuario.
    • Almacenamiento inseguro: datos en LocalStorage sin cifrar son vulnerables a ataques XSS.
    • Mitigación: consentimiento granular, anonimización, HTTPS obligatorio, retención mínima y borrado del perfil en menos de 72 horas.
